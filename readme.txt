# 原片实时库存系统 (Glass Inventory Management System)

## 项目概述

**系统名称**: 原片实时库存系统  
**版本**: 1.0.0  
**开发语言**: PHP + MySQL + HTML/CSS/JavaScript  
**适用场景**: 玻璃制造企业的原片库存管理  
**支持设备**: PC端管理 + 移动端操作  

## 系统特点

- **多基地管理**: 支持多个生产基地的独立库存管理
- **实时库存**: 实时跟踪原片包的流转状态和库存数量
- **移动端支持**: 支持PDA/手机扫码操作，适合仓库现场使用
- **权限管理**: 多角色权限控制（管理员、库管、操作员、查看者）
- **数据追溯**: 完整的操作记录和流转历史追踪
- **报表统计**: 每日领用总表和各类统计报表

## 核心功能模块

### 1. 基础数据管理
- **基地管理**: 多基地信息维护
- **库位架管理**: 库存区、加工区、报废区的库位架配置
- **原片类型管理**: 原片规格、品牌、生产商等基础信息
- **用户管理**: 用户账号、角色权限管理

### 2. 库存操作
- **采购入库**: 新采购原片包入库登记
- **领用出库**: 整包或部分领用到加工区
- **归还入库**: 加工后剩余原片归还库存
- **报废处理**: 不合格原片的报废处理
- **库存盘点**: 库存盘盈盘亏处理

### 3. 移动端功能
- **扫码操作**: 支持二维码/条码扫描
- **快速流转**: 移动端快速完成库存流转操作
- **实时查询**: 移动端实时查询包信息和库存状态

### 4. 报表统计
- **每日领用总表**: 按日期统计各基地领用情况
- **库存统计**: 实时库存数量和面积统计
- **流转记录**: 详细的操作历史记录
- **数据导出**: 支持Excel格式数据导出

### 5. 系统管理
- **系统设置**: 基本参数配置
- **操作日志**: 用户操作记录追踪
- **数据备份**: 数据库备份和恢复

## 技术架构

### 后端技术栈
- **PHP**: 7.4+
- **MySQL**: 5.7+ / 8.0+
- **Apache/Nginx**: Web服务器

### 前端技术栈
- **HTML5/CSS3**: 响应式界面设计
- **JavaScript/jQuery**: 交互功能实现
- **Bootstrap**: UI框架
- **ZXing**: 移动端扫码功能

### 数据库设计
- **核心表**: 8个主要业务表
- **视图**: 2个统计汇总视图
- **索引优化**: 针对查询性能优化
- **外键约束**: 保证数据完整性

## 核心数据表结构

### 主要业务表
1. **bases** - 基地信息表
2. **storage_racks** - 库位架信息表
3. **glass_types** - 原片基础信息表
4. **glass_packages** - 原片包信息表
5. **inventory_transactions** - 库存流转记录表
6. **inventory_operation_records** - 库存操作记录表（新设计）
7. **users** - 用户表
8. **settings** - 系统设置表

### 辅助表
- **dictionary_items** - 数据字典表
- **transaction_logs** - 操作日志表

### 视图
- **daily_operation_summary** - 每日操作汇总视图
- **package_operation_history** - 包操作历史视图

## 业务流程

### 1. 采购入库流程
采购到货 → 扫描包号 → 选择库位架 → 确认入库 → 更新库存
### 2. 领用出库流程
扫描包号 → 选择目标区域 → 输入领用数量 → 确认出库 → 更新状态
### 3. 归还入库流程
扫描包号 → 确认归还数量 → 选择库位架 → 确认归还 → 更新库存
### 4. 报废处理流程
扫描包号 → 选择报废原因 → 确认报废 → 更新状态为已报废


## 权限角色说明

- **admin（管理员）**: 系统全部功能权限
- **manager（库管）**: 库存管理、报表查看权限
- **operator（操作员）**: 基本库存操作权限
- **viewer（查看者）**: 仅查看库存和报表权限

## 安装部署

### 环境要求
- PHP 7.4+
- MySQL 5.7+ 或 8.0+
- Apache 2.4+ 或 Nginx 1.18+
- 支持PHP扩展：mysqli, json, session

### 部署步骤
1. 将项目文件上传到Web服务器目录
2. 创建MySQL数据库并导入database_new.sql
3. 修改config/database.php中的数据库连接信息
4. 设置temp/和backups/目录的写入权限
5. 访问系统进行初始化配置

### 默认账号
- 用户名：admin
- 密码：admin123
- 首次登录后请及时修改密码

## 系统特色功能

### 1. 智能库存追踪
- 实时更新包的片数和状态
- 支持部分领用的精确记录
- 自动计算实际领用量和剩余量

### 2. 移动端优化
- 适配PDA设备分辨率（720x1280）
- 支持多种扫码方式（摄像头、输入）
- 离线缓存关键数据

### 3. 数据完整性
- 事务处理保证数据一致性
- 完整的操作日志记录
- 支持操作撤销和数据恢复

### 4. 报表分析
- 按工作时间段（8:00-次日8:00）统计
- 支持多维度数据分析
- 实时生成各类统计报表
